Kubernetes Cluster with Linode

February 20, 2021

Concepts

Often, we need a kubernetes cluster for our lab/spike. This could be done with several ways without needing any real cluster. The following are some alternatives for local clusters.

  • minikube Local kubernetes cluster, limited to one node and relies on virtualization.
  • kind Kubernetes in Docker (that is why it called KinD). Used by kubernetes maintainer for their testing. A lot of very friendly developer tool.
  • microk8s Lightweight multi-node cluster. Pretty similar to minikube, it spins up cluster with its add-ons.
  • k3s/k3d Lightweight k8s distro designed for developers/IoT device.
  • firekube Kubernetes cluster using ignite and firecracker for lightweight VMs

As much as I like them, often running them on less powerful machine is just unbearable, especially when working on many containers/devops tooling.

Setting up EKS is just too costly for a personal lab, mainly because AWS charged $0.10 for the cluster, and its nodes charged separately.

What I Ended up Doing

I needed an alternative, something simple and affordable.

I narrowed down my search to DigitalOcean and Linode. Here is what I think (in a context of getting this for my lab, not a production).

  • DigitalOcean Pretty easy to set up, I really like their How-Tos and API documentation. They have a lot of handy features whenever you need it for any product. Their API is really helpful if you need to programmatically control them.
  • linode A very simple UI, you could set up a cluster with few clicks. The way you bind the cluster to the node is really easy. At the time I wrote this, I got $100 balance to try out their cluster. When I write this post, I would prefer the way DigitalOcean structured their documentation but I really like their simplicity.

Long story short, I picked up Linode over DO for my lab.

How it looks like

Below is their UI when I try to create the cluster, simple and intuitive.

In one page, they ask you the cluster name, region, kubernetes version, nodes you wanted to add to the cluster. They display very clear pricing.

For the cluster that created. It provides the link to the node configuration and link to your kubernetes configuration. Pretty handy.

After I pasted my config to my ~/.kube/configs, everything is ready.

Conclusion

Don’t get me wrong, I really like DigitalOcean, I have been using it for many years but for the sake of my lab, simplicity and affordability, I’ll give linode a go.
